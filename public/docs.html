01

What Moon does

Moon is a lightweight music streaming app that behaves like a native one. Every song is searchable, playable instantly, and can be kept for later in one of two ways — inside the app's own library, or as a real file on the device. The interface stays quiet and dark, so nothing distracts from what's playing.

Search & play

Debounced search against the catalog, with trending and pinned picks on the home screen.

Save in Moon

Downloads the track into IndexedDB. Plays back offline from Moon's own library — no OS file involved.

Save to device

Writes a real MP3 to the phone's Downloads folder, with cover art embedded via ID3 tags.

Works offline

The app shell itself — not just saved songs — precaches on first load, so it opens with no connection at all.

04

How offline actually works

There are two separate systems here, and it matters to keep them apart.

1

The app shell — cached by the service worker

On first visit, src/sw.js precaches every built JS, CSS, and HTML file using Workbox's precacheAndRoute. This is what lets Moon open at all with no connection — even in a fresh tab that's never been used before.

2

Your music — stored in IndexedDB

Songs saved via "Save in Moon" live in downloadDB.js, entirely separate from the service worker. This is why downloaded tracks keep playing offline regardless of shell-caching state.

npm run build
npm run preview

Service workers only behave predictably in a production build — not in npm run dev. Always test offline behavior against preview, not dev.

Getting started

1

Install dependencies

npm install
2

Run locally

npm run dev

Opens at localhost:5173. Service worker features won't precache reliably here — that's expected.

3

Build & verify offline

npm run build
npm run preview

Check DevTools → Application → Service Workers, then toggle Network → Offline and reload.
